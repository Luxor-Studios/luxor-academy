**Example: JIT Compiler with Timing-Preserving Verification**

// Novel: Prove JIT compilation preserves timing behavior
type timing_trace = list nat

// Define operational semantics with timing
val eval_with_timing: expr -> Tot (value * timing_trace)

// JIT compilation preserves observable timing
val jit_compile_timing_preserving:
  e: expr ->
  Lemma (snd (eval_with_timing e) ==
         snd (eval_with_timing (jit_compile e)))

05Self-Check

1 — Per the fstar-framework's L7 definition, what distinguishes Level 7 from Level 6?
The spec reads L6 (established techniques) ⊂ L7 (established + novel). L6 exhausts the established playbook — semantic modelling, relational reasoning, categorical laws. L7 is the rung where you invent a construction the base framework does not already have: a custom effect like JitST, a novel indexed family, a new semantic model. Everything L1-L6 is a prerequisite; L7 is the extension.
2 — The JitST effect in the L7 example is defined atop which earlier-level primitive?
effect JitST (a: Type) (pre: ...) (post: ...) (time: nat) = ST a pre (fun h0 x h1 -> post h0 x h1 /\ timing_cost h0 h1 <= time). It is a **refinement of ST** — the L4/L5 stateful effect — adding a fourth parameter (the polynomial time bound) and conjoining a timing predicate to the postcondition. L7 extends L5; the novel part is the extra index, not a rewrite from scratch.
