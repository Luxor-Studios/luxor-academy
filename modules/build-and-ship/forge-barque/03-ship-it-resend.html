01Primer

Transactional mail without the SMTP detour

BARQUE does not speak SMTP directly. It shells out to Charm Pop, a single-binary CLI that already knows how to talk to Resend's HTTP API or fall back to SMTP on request. The integration lives in barque/core/email.py.

The contract is simple and mostly environmental: set RESEND_API_KEY in your shell, pick a verified --from address, and call barque send doc.md --to someone@example.com. BARQUE builds a pop command line (--from, --to, --subject, --attach), injects RESEND_API_KEY into the subprocess environment via _get_env_vars(), and hands off. Pop takes it from there.

Why Resend instead of your company's SMTP relay? Two reasons, both load-bearing for a solo builder. First, SMTP credentials are the highest-value secret on your machine — Resend lets you rotate a narrow-scope API key instead. Second, Resend ships a free tier that only delivers to your verified email (BARQUE's README.md calls this out explicitly). That constraint is a feature: it makes Resend safe as a development dependency without a domain verification ritual, and you can promote to full-domain delivery by verifying at resend.com/domains when you are ready to ship for real.

BARQUE treats email as a subprocess, not a protocol. Pop wraps Resend's HTTP API behind --from/--to/--subject/--attach, and the only secret that crosses the process boundary is RESEND_API_KEY.

    def _get_env_vars(self) -> Dict[str, str]:
        # L227: copy — never pass by reference; subprocess should not inherit parent mutations
        env = os.environ.copy()

        # L230–232: the Resend branch. Key goes into env, never into argv.
        if self.config.provider == EmailProvider.RESEND and self.config.resend_api_key:
            env["RESEND_API_KEY"] = self.config.resend_api_key

        # L235–243: the SMTP branch. Four variables; all POP_SMTP_* prefixed, read by pop.
        if self.config.provider == EmailProvider.SMTP:
            if self.config.smtp_host:
                env["POP_SMTP_HOST"] = self.config.smtp_host
            ...

        # L246–251: sender identity. POP_FROM is read by pop as the From: header default.
        if self.config.from_email:
            env["POP_FROM"] = self.config.from_email
        if self.config.signature:

        return env
Why env vars, not flags? Command-line arguments appear in ps listings, shell history, and log files. Environment variables of a subprocess do not. BARQUE deliberately routes every secret through env= so RESEND_API_KEY never becomes visible to other users on the machine.

    def _get_env_vars(self) -> Dict[str, str]:
        """Get environment variables for Pop"""
        import os
        env = os.environ.copy()

        # Resend API key
        if self.config.provider == EmailProvider.RESEND and self.config.resend_api_key:
            env["RESEND_API_KEY"] = self.config.resend_api_key

        # SMTP configuration
        if self.config.provider == EmailProvider.SMTP:
            if self.config.smtp_host:
                env["POP_SMTP_HOST"] = self.config.smtp_host
            if self.config.smtp_port:
                env["POP_SMTP_PORT"] = str(self.config.smtp_port)
            if self.config.smtp_username:
                env["POP_SMTP_USERNAME"] = self.config.smtp_username
            if self.config.smtp_password:
                env["POP_SMTP_PASSWORD"] = self.config.smtp_password

        # From email
        if self.config.from_email:
            env["POP_FROM"] = self.config.from_email

        # Signature
        if self.config.signature:
            env["POP_SIGNATURE"] = self.config.signature

        return env

05Self-Check

1 — Per barque/core/email.py, where does RESEND_API_KEY live at the moment BARQUE invokes the pop subprocess?
email.py lines 231–232: env["RESEND_API_KEY"] = self.config.resend_api_key. The key is injected into the subprocess environment via the env= argument of subprocess.run (line 121). It is never passed as a command-line flag, where it would appear in process listings.
2 — BARQUE's README.md Email Setup section warns that free Resend accounts have a specific delivery constraint. What is it?
README.md line 48: "Free Resend accounts can only send to your verified email (the one you signed up with). To send to any recipient, verify a domain at resend.com/domains." This is why BARQUE's default recipient in many configs is the author's own address — it works out of the box on a free tier.
3 — If you set --provider smtp instead of the default, which of these environment variables does BARQUE set for the Pop subprocess?
email.py lines 236–243: the SMTP branch sets POP_SMTP_HOST, POP_SMTP_PORT, POP_SMTP_USERNAME, POP_SMTP_PASSWORD. RESEND_API_KEY is only set when provider == EmailProvider.RESEND, so choosing SMTP swaps the entire secret surface.
