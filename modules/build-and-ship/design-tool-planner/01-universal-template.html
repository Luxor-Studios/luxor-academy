01Primer

One template, any visual API, any AI generator

The Universal Design Tool Planning Template is not a Canva workflow wearing a disguise. It is a meta-pattern — a shape you fill in with your own API names, your own quality targets, and your own user personas. UNIVERSAL-TEMPLATE-README.md calls out the move in its second paragraph: the same five-step pipeline (analyze → research → architect → plan → document) applies whether you are wiring Claude to Figma plus DALL-E, to Sketch plus Stable Diffusion, or to Adobe Express plus Midjourney. The APIs change. The skeleton does not.

The skeleton is parameterised by seven variables. Four are required (PROJECT_NAME, PRIMARY_API, SECONDARY_SERVICES, TARGET_USERS) and three are optional with defaults (SPEED_TARGET: < 30s, QUALITY_TARGET: 90%+, ARCHITECTURE_PATTERN: two-agent). Pick values that describe your project, and the workflow produces the same six documents every time: PROJECT-OVERVIEW.md, ARCHITECTURE.md, API-INTEGRATION.md, WORKFLOWS.md, README.md, PROJECT-SUMMARY.md.

Why does this matter? Because the costly part of planning a design-automation tool is not choosing the API — it is structuring the decision. The template freezes the structure so you can spend your thinking on the variables that actually vary. The CANVA-CODE validation run proved the shape: 22 minutes, 9.18/10 average quality, 91% token efficiency, 3,822 lines of implementation-ready docs. Swap in Figma and DALL-E and you get the same shape.

The APIs change. The skeleton does not. Seven variables parameterize every design-automation project; the five-step pipeline is universal.

### Required Variables

```yaml
PROJECT_NAME: "Your project name"
  # Examples: "FigmaAI", "CanvaBot", "SketchAssist"

PRIMARY_API: "Your main visual API"
  # Options: Canva, Figma, Adobe Express, Sketch, InVision, Miro
  # Determines: Visual scaffolding capabilities

SECONDARY_SERVICES: "Your AI generation service(s)"
  # Options: NanoBanana, DALL-E, Midjourney, Stable Diffusion, Leonardo.ai
  # Determines: Image/icon/design generation quality and speed

TARGET_USERS: "Your designer personas"
  # Examples: product designers, marketing teams, content creators, freelancers
  # Determines: Workflow design and success metrics
```

### Optional Variables

```yaml
SPEED_TARGET: "Your performance goal"
  # Default: < 30s
  # Options: < 15s (fast), < 30s (balanced), < 60s (quality)

QUALITY_TARGET: "Your quality threshold"
  # Default: 90%+
  # Options: 85%+ (draft), 90%+ (production), 95%+ (premium), 99%+ (enterprise)

ARCHITECTURE_PATTERN: "Your agent design"
  # Default: two-agent
  # Options: single-agent, two-agent, multi-agent
```
