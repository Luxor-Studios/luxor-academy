01Primer

The DSL that lets you write prompt compositions like Unix pipes

The meta-prompting-composition skill (CMP Query Language) gives you a small, typed DSL for saying what you want at a high level and having it compile down, through several layers, to actual LLM calls. At the top layer you write a pipeline like ASSESS → TRANSFORM → RMP^3 → OUTPUT FOR "optimize database queries". At the bottom layer, that compiles to /blocks (atomic block composition), which compiles to /meta / /rmp command invocations, which finally execute prompts.

The grammar is deliberately tiny: a pipeline is a sequence of stages joined by the four categorical operators you already know — → (sequence), || (parallel), >=> (Kleisli, quality-gated), ⊗ (tensor). Stages are either macros (uppercase, expand to block compositions: ASSESS, TRANSFORM, EXECUTE, QUALITY, REFINE, CONVERGE, ITERATE, RMP, FULL), atomic blocks (lowercase: assess_difficulty, select_tier, execute_prompt, etc.), or parenthesized groups. ITERATE expands to EXECUTE → QUALITY >=> REFINE — a single RMP iteration — and RMP^3 repeats that three times.

Why a DSL instead of hand-written commands? Because composition needs to be typed to be trustworthy. The CMP language is parsed, validated against the macro expansion table, and compiled to commands whose categorical laws are already verified. If you write TRANSFORM → ITERATE^3 FOR "build robust API", the compiler produces /rmp @quality:0.8 @max_iterations:3 "build robust API" — and the mathematical guarantee (monad associativity, Kleisli improvement law) propagates from the generated command back up to your pipeline. Unix pipes let you compose grep | awk | sort; CMP lets you compose ASSESS | TRANSFORM | RMP — with quality algebra instead of byte streams.

CMP is a small typed DSL whose four operators are exactly the four categorical composition operators. High-level macros compile down to verified block pipelines — so the mathematical laws your pipeline inherits come from the compiler, not from your care.

05Self-Check

1 — According to the CMP macro expansion table, the ITERATE macro expands to what sequence of atomic blocks?
ITERATE → [execute_prompt → assess_quality >=> extract_improvement → apply_refinement]. This is the canonical form of a single RMP iteration: execute, then quality-gate (>=> Kleisli — only proceed if quality is below threshold), then extract and apply refinement. See meta-prompting-composition/skill.md §Macro Expansion Table.
2 — In the CMP grammar, which operator indicates a quality-gated Kleisli composition that only proceeds if the preceding stage's quality is below threshold?
>=> is the Kleisli operator. Its semantics (from the framework): at each stage, evaluate quality; if below threshold, refine before moving on. Distinct from → (plain sequence, no gate), || (parallel), and ⊗ (tensor). This is the operator that ITERATE uses between assess_quality and extract_improvement → apply_refinement.
3 — Why does CMP compile through four layers (Query Language → Blocks → Commands → Prompts) instead of generating LLM prompts directly?
Each intermediate layer has independently verified properties. The atomic block layer guarantees the composition laws hold; the command layer guarantees the modifiers (@quality, @tier, @mode) parse correctly; the prompt layer is the only place with LLM non-determinism. Keeping the layers separate means a failure at the LLM layer cannot corrupt the algebraic guarantees above it. It is the same reason Unix pipes do not let awk rewrite your shell grammar.
