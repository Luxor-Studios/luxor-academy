Recursive Meta-Prompting: The Fixed Point

// Tier: Master · Source: categorical-meta-prompting/.claude/skills/recursive-meta-prompting/skill.md

01Primer

A prompt that rewrites itself until it stops changing

Recursive Meta-Prompting (RMP) is the framework's name for quality-gated iteration: you start with a prompt p₀, evaluate its quality q₀ ∈ [0,1], and if q₀ is below your threshold θ, you apply an improvement morphism to get p₁, evaluate q₁, and continue. You stop when either qₙ ≥ θ (success) or you hit max_iterations (give up). The categorical reading of this loop is exactly a fixed-point computation: you are searching for a p* such that improve(p*) = p* — the prompt that does not change any more, because the quality threshold has been met.

Structurally, RMP is the Monad M of the framework: M: Prompt →ⁿ Prompt, with return lifting a raw prompt into the refinement context, >>= (bind) chaining an improvement step, and join flattening a nested refinement. The three monad laws — left identity (return >=> f = f), right identity (f >=> return = f), and associativity ((f >=> g) >=> h = f >=> (g >=> h)) — are what guarantee that rearranging the order in which you group refinement stages does not change the final prompt. The recursive-meta-prompting skill notes that all 15 of its monad-law tests pass.

There is a second structure here: Comonad W, which handles the history half of the loop. extract: W(a) → a returns the current best prompt; duplicate: W(a) → W(W(a)) gives you the history-of-histories so you can observe why quality plateaued; extend: (W(a) → b) → W(a) → W(b) applies a history-aware refinement. The monad writes forward; the comonad reads backward. Together they give RMP its two essential moves: refine with memory, converge on a fixed point. That is exactly what /rmp @quality:0.85 "optimize algorithm" does when you invoke it.

RMP is a fixed-point operator on the category of prompts: iterate improve() until improve(p*) = p*. The Monad M provides the iteration; the Comonad W provides the history-awareness that lets you detect plateau versus real progress.

Fixed-Point Semantics

RMP treats prompt improvement as a fixed-point computation:

  F: Prompt → Prompt
  Fix(F) = p where F(p) = p (converged prompt)

The monad structure ensures:
- Return: Initial prompt enters refinement context
- Bind (>=>): Quality assessment chains to improvement
- Join: Nested refinements flatten to single improvement

05Self-Check

1 — RMP is described categorically as a 'fixed-point computation'. Which equation defines the fixed point p* that the loop is searching for?
The fixed point is p* such that improve(p*) = p* — applying the improvement morphism one more time does not change the prompt. In RMP this is operationalised as quality(p*) ≥ θ, at which point further iteration would either not change p or exceed budget. The skill states: Fix(F) = p where F(p) = p (converged prompt).
2 — Which of the following is NOT one of the three monad laws that RMP is required to satisfy?
The three monad laws are left identity (return >=> f = f), right identity (f >=> return = f), and associativity ((f >=> g) >=> h = f >=> (g >=> h)). Commutativity (f >=> g = g >=> f) is NOT a monad law — monads are deliberately non-commutative because order of effects matters (refining-then-evaluating is not the same as evaluating-then-refining). Confusing associativity with commutativity is a classic slip.
3 — The RMPContext comonad exposes a quality_trend() method that classifies the history as RAPID_IMPROVEMENT, STEADY_IMPROVEMENT, PLATEAU, or DEGRADING. Why does RMP need a comonad for this, rather than just a monad?
Monads write forward (accumulate effects, produce the next prompt). Comonads read backward (extract patterns from the history). Detecting PLATEAU vs STEADY_IMPROVEMENT requires looking at the entire quality trajectory q₀, q₁, ..., qₙ — that is a co-Kleisli arrow W(History) → Classification, not a Kleisli arrow. The skill's RMPContext class uses duplicate to access the full quality history and extend to apply a history-aware transformation.
