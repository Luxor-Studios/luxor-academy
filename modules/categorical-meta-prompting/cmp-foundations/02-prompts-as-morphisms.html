01Primer

Prompts are not strings. Prompts are typed arrows.

The first move of categorical-meta-prompting is to stop treating a prompt as String and start treating it as a morphism — a typed arrow between context objects. In the framework's central functor F: Task → Prompt, a task is an object with structure (keywords, modifiers, operators), and a prompt is an object with structure ({context}, {mode}, {format}, assembled body). F is the structure-preserving map between them.

The guide decomposes F further: F = F_template ∘ F_tier ∘ F_domain. Raw task keywords flow through a domain classifier, then a tier selector (L1..L7 complexity), then a template assembler. Each step is its own arrow. Composition means: running the three sub-functors in order produces the same prompt as running the composite F — that is exactly the composition law F(g ∘ f) = F(g) ∘ F(f) applied to your concrete pipeline.

Why this buys you something: once prompts are arrows between typed objects, you get composition for free. A → B → C means run the first prompt whose output becomes the input context for the second, whose output becomes input for the third, and the quality of the composite is min(q_A, q_B, q_C) — the weakest link. Parallel prompts A || B aggregate as mean. You never have to invent this bookkeeping; it falls out of the algebra. The /meta command is the executable form of F; when you type /meta "implement rate limiter", the framework is quite literally evaluating a functor application.

A prompt is a morphism between typed context objects, not a string. Composition, identity, and quality-tracking then fall out of the algebra instead of being bolted on.

05Self-Check

1 — The guide decomposes F: Task → Prompt as F_template ∘ F_tier ∘ F_domain. If composition is associative, which of the following rewrites is guaranteed to produce the same prompt?
Associativity says (F_template ∘ F_tier) ∘ F_domain = F_template ∘ (F_tier ∘ F_domain). Both groupings produce the same prompt because the framework requires F to satisfy F(g ∘ f) = F(g) ∘ F(f) (CATEGORICAL-ONTOLOGY.md §2.1). The parentheses only change evaluation order, not the result. Reversing or permuting the functors changes the meaning because composition is NOT commutative.
2 — You compose two prompts sequentially: A → B where q(A) = 0.9 and q(B) = 0.7. What is the quality of the composite under the framework's rule?
Sequential composition uses min: q(A → B) = min(q_A, q_B) = min(0.9, 0.7) = 0.7. This is stated in docs/CATEGORICAL-META-PROMPTING-COMPLETE-GUIDE.md §Composition Operators. Averaging (0.8) would be the rule for || parallel composition, not sequence.
3 — Why does treating a prompt as a string (rather than a typed morphism) break the composition law?
When every value is String, every arrow is String → String and there is no type-level constraint on what can compose with what. The composition law F(g ∘ f) = F(g) ∘ F(f) becomes vacuous because there is nothing distinguishing f from g. Typed objects (Task, Prompt, Context) make the law a real, load-bearing structural claim — without them, composition degenerates to concatenation. (String concatenation IS associative, by the way, which makes (C) a trap.)
