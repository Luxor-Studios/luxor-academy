01Primer

Install, restart, verify — then register one tool

A working MCP server is a three-step loop, and the meta-prompting-plugin's docs/QUICKSTART.md shows the shape of that loop cleanly for any Claude Code extension: clone, run INSTALL.sh, restart Claude Code, then verify with a single command. An MCP server follows exactly that rhythm — you scaffold, you wire it into the client, you restart, and you send one request to confirm the handshake succeeded.

The TypeScript scaffold is three concrete moves. First, construct a Server with an { capabilities: { tools: {} } } declaration (Module 02 already taught you why). Second, register a ListToolsRequestSchema handler that returns one analyze tool with a zod-compatible inputSchema. Third, register a CallToolRequestSchema handler that pattern-matches on the tool name and returns content: [{ type: "text", text: ... }]. Bind a StdioServerTransport, call server.connect(transport), and you are done — that is a complete, shippable server in roughly thirty lines.

The QUICKSTART verification step translates directly: you wire the server into Claude Code's ~/.claude.json, restart the client, and run a single test command. If the client lists your tool, your initialize handshake worked, your capability bag was accepted, and your ListToolsRequestSchema handler fired. Three concentric contracts — transport, handshake, schema — all passed in a single round-trip.

Every MCP server ships in the same three-step loop: install / restart / verify. If listTools returns your registered tool, all three protocol layers are working.

# Quickstart

Get productive in 5 minutes.

## Install

```bash
git clone https://github.com/manutej/meta-prompting-plugin ~/.claude/plugins/meta-prompting
cd ~/.claude/plugins/meta-prompting
./INSTALL.sh
```

Restart Claude Code.

## Verify

```
/meta --help
```

You should see the command frontmatter. If not, check that `~/.claude/commands/meta.md` is a symlink pointing into `~/.claude/plugins/meta-prompting/commands/meta.md`.

05Self-Check

1 — docs/QUICKSTART.md gives a three-step install loop for any Claude Code extension. Which step comes immediately after running the installer?
QUICKSTART.md line 13: Restart Claude Code. Extensions are only loaded at client startup; skipping the restart fails the verify step every time.
2 — In the mcp-categorical TypeScript scaffold, which handler does the client invoke during the initialize verification round-trip to enumerate available tools?
mcp-categorical SKILL.md lines 37–52 register ListToolsRequestSchema; the client sends tools/list after the handshake and receives the tool array. CallToolRequestSchema fires later, on actual invocation.
3 — The analyze tool in the skill's example returns its output as content: [{ type: "text", text: "Analyzed: ..." }]. Why the content array rather than a bare string?
Typed content parts let a single tool return text + image + resource together. A bare string would collapse that structure.
