Wire It Into Claude Code

// Tier: Adept · Source: meta-prompting-plugin/docs/EXAMPLES.md

01Primer

Register once; invoke like any other tool

A server is only useful when a client knows how to find it. Claude Code has one command for that — claude mcp add <name> -- <launch-command> — and one config file it writes to: ~/.claude.json. The entry stores a human name, the process launch line (often node dist/server.js for TypeScript or python -m my_mcp for Python), and an optional env block for anything secret. That is the whole registration surface.

Once registered, the server participates in a Claude Code session exactly the way a /meta → /rmp → /review pipeline does in docs/EXAMPLES.md: the client composes the server's capabilities into the chain. When Claude decides a tool is appropriate, it sends tools/call down the registered transport, receives the typed result, and splices it into the conversation. The wiring is declarative at setup, automatic at runtime.

When the transport refuses to speak — and it will, the first time — the debug loop is small and mechanical. Check claude mcp list to confirm the server is registered. Check the process is actually launchable from the exact path in ~/.claude.json (absolute paths beat shell aliases every time). Tail the server's stderr while Claude Code boots to catch initialize failures. If the handshake completes but no tools appear, your ListToolsRequestSchema handler is not registered — or your capabilities bag never mentioned tools. Three places to look; one of them is always the cause.

claude mcp add is one command that writes one JSON entry into ~/.claude.json. When the handshake fails, the cause is almost always the launch path, the capability bag, or the handler registration — in that order.

## 1. Ship a feature end-to-end

**Goal:** implement a feature, iterate until good, then review before merge.

```
/chain [/meta→/rmp→/review] @quality:0.9 "add rate limiting to /api/login with exponential backoff"
```

- `/meta` picks a strategy based on complexity (likely L4 — algorithm + integration).
- `/rmp` iterates until the implementation scores ≥ 0.9 on the quality rubric.
- `/review` does a final safety pass.

Chain quality rule: `min(q_meta, q_rmp, q_review)` — weakest link gates the output.

05Self-Check

1 — Which single CLI command registers an MCP server with Claude Code?
The command writes an entry into ~/.claude.json's mcpServers map and records the launch line Claude Code should spawn on startup.
2 — In docs/EXAMPLES.md §1, the /chain quality rule is min(q_meta, q_rmp, q_review). Why is that the right gate for a client-integrated MCP server too?
EXAMPLES.md line 19: weakest link gates the output. For MCP-integrated chains, a broken handshake or missing capability is a weak link that must gate the whole pipeline.
3 — Claude Code completed initialize but no tools appear when the model asks for them. Which of these is NOT a plausible cause?
If ~/.claude.json were missing the handshake would never have happened. The real causes are capability-bag omission, handler not registered, or handler returning an empty array.
