Primer

Publish the server; submit the install command

You have built a server, wired it into Claude Code, and watched it answer a tools/call. The last step is the one that matters to anyone but you: turning your local binary into something a stranger can install with one command.

The shape to imitate is the meta-prompting-plugin's own README.md. Look at its roadmap: v0.1.0 is a git-clone install, v0.2.0 promises a TypeScript library and an MCP server, and v0.3.0 aims at the Claude Code marketplace. That is the arc every server author walks. You start as git clone && ./INSTALL.sh, you harden into a versioned package on npm or PyPI, and — when the marketplace accepts your submission — you reduce to a one-liner install: claude mcp add <your-server>.

The sovereignty clause of this Quest is the same one Forge BARQUE ends on. You do not finish holding a certificate; you finish holding a URL and an install command that teaches the next learner what you built. Submit that install command as your Quest proof. In Phase 1 of LUXOR Academy the submission is captured locally and queued for the codebase-to-course pipeline. In Phase 3 the pipeline runs live: it extracts your README, your capability declaration, and one representative tool handler, validates them against validators/slots.schema.json, and mints a permalink Quest that teaches your MCP server by the same six-slot contract you just completed.

You came in knowing what MCP was. You leave with a server other people can install.

    • Q2. What distinguishes a v0.3.0-style marketplace install from the v0.1.0 git-clone install?

        Show explanation

        A marketplace install collapses to a single claude mcp add <name> line that the client resolves from its registry. Git-clone installs require the user to pick a checkout path, run the installer, and wire the launch line by hand — three steps where the marketplace needs one.

      • Q3. Why is the canonical last module of every LUXOR Academy Quest a 'Ship Your Own' step rather than a summary quiz?

          Show explanation

          Sovereignty. The Academy's canonical-last-module pattern requires learners to leave each Quest holding a permalink to their own work — an install command, a repo URL, a published server — not a score. A summary quiz trains for the Quest. Ship-Your-Own trains for the next repo.
