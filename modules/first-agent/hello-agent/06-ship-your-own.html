The five modules before this one handed you every part of an agent: a package and key (01), a persona and a tool schema (02), a token-by-token renderer (03), a while (true) loop keyed on stop_reason (04), and a cache_control annotation that makes the loop affordable (05). There is nothing left to teach about how to build an agent. What is left is to wrap the thing in a CLI shell and publish it.

The cookbooks/RESEARCH-AGENT.md example on lines 222-262 shows the archetype: a small class (ResearchOrchestrator) that composes three agent helpers (PlanningAgent, ResearchAgent, SynthesisAgent), exposes one public method (conductResearch(topic)), and runs from a top-level async function main() that reads from process.env.ANTHROPIC_API_KEY and prints the result. That is a shippable CLI. Rename the class, swap the helpers for your own, add a #!/usr/bin/env node shebang plus a bin entry in package.json, and you have a published command.

class ResearchOrchestrator {
  private planner: PlanningAgent;
  private researcher: ResearchAgent;
  private synthesizer: SynthesisAgent;

  constructor(apiKey: string) {
    this.planner = new PlanningAgent(apiKey);
    this.researcher = new ResearchAgent(apiKey);
    this.synthesizer = new SynthesisAgent(apiKey);
  }

  async conductResearch(topic: string): Promise<string> {
    console.log(`Planning research for: ${topic}`);

    // Step 1: Plan
    const subtopics = await this.planner.planResearch(topic);
    console.log(`Identified ${subtopics.length} subtopics`);

    // Step 2: Research (parallel)
    console.log("Conducting research...");
    const researchPromises = subtopics.map((subtopic) =>
      this.researcher.research(subtopic)
    );
    const results = await Promise.all(researchPromises);

    // Build findings map
    const findings: Record<string, string> = {};
    subtopics.forEach((subtopic, i) => {
      findings[subtopic] = results[i];
    });

    // Step 3: Synthesize
    console.log("Synthesizing findings...");
    const report = await this.synthesizer.synthesize(topic, findings);

    return report;
  }
}
