01Primer

From zero to a Claude reply in three lines of shell

Before an agent can loop, stream, or call tools, it has to exist. The Claude Agent SDK compresses that first-existence step into a very small ritual: install one package, export one key, run one message. The vendored multiplatform skill lays out the exact commands in its README — one block per language (npm install @anthropic-ai/sdk, pip install anthropic, go get github.com/anthropics/anthropic-sdk-go), then a twin block that calls client.messages.create({ model, max_tokens, messages }) and prints the reply. That is the whole scaffold.

Two boundaries matter here. The first is the API key boundary: the SDK reads ANTHROPIC_API_KEY from the process environment, never from a hardcoded string. Put the key in .env, load it with dotenv, and your scaffold is safe to push. The second is the model boundary: you pass a model ID (the skill's canonical default is claude-sonnet-4-5-20250929) and a max_tokens cap. Those two values are the knobs for cost and latency for every subsequent module in this Quest.

What the scaffold intentionally does not give you yet: a system prompt, a tool, a stream, a loop. All it gives you is a round-trip. Once that round-trip works — once console.log(message.content) prints actual Claude output — you have the bedrock every later module builds on. Skip this and nothing else compiles.

An agent scaffold is three things: a package, a key in the environment, and one messages.create call. Everything else in this Quest is an elaboration of that round-trip.
