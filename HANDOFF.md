## 3 · Architecture (v2 — see `docs/ARCHITECTURE-v2.md` for full decision record)

Two layers, one design system:

- **Shell** (`web/`): Next.js 15 App Router + shadcn/ui + Tailwind v4, **static export** (honors prior "no runtime SSR" invariant). Owns: landing, tier pages, progress dashboard, atlas hub, XP sovereignty, navigation.
- **Modules** (`modules/<track>/<quest>/*.html`): self-contained HTML generated by the `codebase-to-course` skill. Offline-capable, no framework, one file each.
- **Shared design tokens**: `atlas/tokens/tokens.css` imported by shell via Tailwind v4 `@theme` block. Same palette, typography, radii in both layers.

**Invariants preserved from launch-plan-v1.1 (MERCURIO+MARS validated):**

1. Tokens-only theming (no runtime theme JS)
2. Three fonts max (Playfair Display + Inter + JetBrains Mono)
3. SVG/CSS only in modules (no WebGL)
4. 6-slot module contract enforced by `validators/six-slots.js`
5. Every widget declares `sandbox_policy` (CSP + iframe sandbox + no-eval)
6. XP export mandatory (sovereignty)
7. `ship_your_own: true` on every Quest's last module
8. AI gates advise, humans decide
9. Every technical claim cited to a file or official doc
